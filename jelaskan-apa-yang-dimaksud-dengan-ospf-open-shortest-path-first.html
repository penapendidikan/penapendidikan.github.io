13. Jelaskan apa yang dimaksud dengan PATH dan CLASSPATH serta bagaimana hubungannya dengan pemrograman Java!


CLASSPATH adalah suatu variabel sistem yang digunakan untuk mengatakan kepada program yang ditulis dengan bahasa Java (termasuk tools yang terdapat dalam SDK) tempat lokasi kelas-kelas yang akan diigunakan.Dengan melakukan setting terhadap nilai variabel CLASSPATH, maka kita dapat melakukan kompilasi maupun eksekusi program yang kita buat didirektori JavaCodeyang terletak di drive D.Apabila terdapat pesan kesalahan pada saat kompilasi terhadap kode-kode program yang terdapat didalam direktori tersebut, maka kita perlu melakukan setting terhadap variabel CLASSPATH dengan perintah berikut (dalam command prompt).
set CLASSPATH=.;D:\JavaCode
Atau apabila anda ingin agar proses kompilasi dan eksekusi program Java dapat dilakukan dari direktori manapun dan bersifat permanen, maka tambahkan direktori binyang terdapat pada Java2SDK ke dalam file autoexec.bat. Berikut contohnya :
set PATH=C:\ProgramFiles\Java\Jdk1.5.0\bin;%PATH%set CLASSPATH=.;D:\JavaCode
Untuk memeriksa setting PATH sudah bekerja atau belum, tuliskan perintah javac atau java disembarang direktori.

30. Klasifikasi routing protokol dibedakan menjadi beberapa jenis, yang bukan termasuk dari klasifikasi routing protokol adalah ....A. Path VectorB. Link State atau Shortest Path FirstC. HybridD. Distance VectorE. Multi Dimensi​


E. Multi Dimensi

Dalam jaringan komputer, routing protokol adalah sekumpulan aturan yang mengatur bagaimana paket data di-routing atau diteruskan melalui jaringan. Routing protokol dapat diklasifikasikan ke dalam beberapa jenis, di antaranya adalah:

Distance Vector: routing protokol yang menggunakan tabel jarak untuk menentukan rute terbaik ke tujuan.Link State atau Shortest Path First (SPF): routing protokol yang menggunakan informasi keadaan (state) dari semua tautan (link) di jaringan untuk menentukan rute terbaik ke tujuan.Path Vector: routing protokol yang menggunakan informasi vektor jalur (path vector) untuk menentukan rute terbaik ke tujuan.Hybrid: routing protokol yang menggabungkan karakteristik dari beberapa jenis routing protokol lainnya.

Dari pilihan jawaban di atas, hanya pilihan E yang tidak termasuk dalam klasifikasi routing protokol, yaitu Multi Dimensi

34. Perbedaan EIGRP dan OSPF yaitu dari segi perangkat EIGRP hanya bisa berjalan atau melakukan routing di perangkat CISCO saja, sedangkan OSPF bisa dijalankan di berbagai perangkat jaringan. Oleh karena itu, OSPF disebut aplikasi routing yang... A. legal B. ilegal C. Open Source D. Close Source E. Remindedjawab dengan benar dan tepat ya :)​


Jawaban:

Jawaban yang tepat adalah C. Open Source.

OSPF (Open Shortest Path First) adalah protokol routing yang dikembangkan sebagai standar terbuka (open standard) yang dapat diimplementasikan pada berbagai perangkat jaringan yang mendukung protokol ini. Oleh karena itu, OSPF dapat dijalankan di perangkat jaringan yang berbeda, termasuk perangkat dari berbagai vendor, yang mendukung protokol OSPF. Sifat "open" dari OSPF mengacu pada fakta bahwa spesifikasi dan dokumentasi protokol ini tersedia untuk umum dan dapat diimplementasikan secara bebas oleh siapa pun yang ingin menggunakannya.
